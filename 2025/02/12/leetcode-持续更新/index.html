1.多数元素

题目需求

给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

示例 1：

题目分析

因为数组中总是存在多数元素。所有题目本质上是寻找数组中出现最多的元素。

思考过程

出现最多的元素，我第一时间想到的方法是使用一个Map映射，记录每个元素出现的次数。但是不符合题目要求空间复杂度为0(1)的要求。（Pass）

使用Boyer-Moore 投票算法

Boyer-Moore 投票算法的核心思想是通过“抵消”机制来找到可能的多数元素：

  1. 假设一个候选元素 candidate，并用一个计数器 count 来记录它的票数。
  2. 遍历数组时：
    • 如果当前元素等于 candidate，则将 count 加 1；
    • 如果当前元素不等于 candidate，则将 count 减 1；
    • 如果 count 变为 0，则更换候选元素为当前元素，并重置 count 为 1。
  3. 最终的 candidate 是潜在的多数元素。
  4. 再次遍历数组，验证该候选元素是否确实是多数元素。

1. 买卖股票的最佳时机

题目需求

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

示例 1：

暴力遍历(超时)

动态规划

思路用一个变量记录历史最低值，用另一个变量记录最大利润

  1. 历史最低值初始化为第一天的价格，最大利润为0
  2. 如果今天价格比历史最低值低，更新历史最低值 (更新买入时机)
  3. 如果当天价格比历史最低值高，(假设今天卖出)比较最大利润和今天假设今天卖出的能获取的利润，更新最大利润。
