FFmpeg使用教程

FFmpeg 是一个开源的多媒体处理工具，它可以用来处理音视频的录制、转换、流式传输、编辑和播放等任务。FFmpeg 支持几乎所有流行的音视频格式，并且提供了一组命令行工具以及一个强大的编程接口，广泛应用于音视频处理的各个领域。

FFmpeg 的主要功能：

  1. 视频转换：支持各种视频格式之间的转换，如 MP4、AVI、MKV、MOV 等。
  2. 音频转换：支持各种音频格式之间的转换，如 MP3、WAV、AAC 等。
  3. 视频编辑：包括剪辑、合并、裁剪、添加水印、旋转、缩放等操作。
  4. 音视频编解码：FFmpeg 提供了对多种音视频编码格式的支持（如 H.264、H.265、VP9、AAC 等）。
  5. 实时流处理：可以进行直播流的推流、拉流等操作，支持 RTMP、RTSP、HLS 等协议。
  6. 视频捕捉与录制：可以从屏幕或设备捕捉音视频。
  7. 视频转码：支持视频的转码（解码和编码），并优化输出文件的大小和质量。

软件下载

1. Windows

使用 FFmpeg 官网下载：

  1. 访问 FFmpeg 官网下载页面：FFmpeg Download
  2. 在“Windows”部分，点击链接进入 FFmpeg Windows builds 或 FFmpeg Windows builds by BtbN。
  3. 下载最新的“Static”版本，例如 ffmpeg-release-essentials.zip（通常包括大多数常用的功能）。
  4. 解压缩下载的 ZIP 文件到一个目录（例如 C:\ffmpeg）。
  5. 将 FFmpeg 的 bin 目录添加到系统的环境变量中：
    • 右键点击“此电脑”或“计算机”，选择“属性”。
    • 点击“高级系统设置” > “环境变量”。
    • 在“系统变量”部分找到 Path，点击编辑。
    • 添加 C:\ffmpeg\bin 到变量值中，然后点击“确定”。
  6. 完成后，打开命令提示符，输入 ffmpeg，如果显示 FFmpeg 版本信息，则表示安装成功。

Redis缓存

缓存穿透

产生原因

请求数据库中没有的数据（被人员观察返回规律攻击）

解决方法

  • 如果数据库没查到数据，redis 也缓存一个空。

  • 通过布隆过滤器（数据结构）有一定误判性

    底层是一个bit map

    将键做多次hash 如果hash值在map中全部命中认为这个键是有效的

    若有一个hash 未命中就认为这个键不存在直接返回不查询redis和数据库

    可以通过Redisson去估算这个map的位数和hash次数

缓存击穿

产生原因

热点key失效

解决方法

  • 热点key永不过期
  • 当 redis 查询失败的时候加一个线程同步锁，只有一个请求可以访问数据库，直到将数据重新写回redis中。

缓存雪崩

产生原因

Redis 中的key集中过期

Redis直接宕机

解决方法

在设置key的过期时间的时候加一个随机数

Redis和数据库数据不一致的问题

修改缓存(x) 删除缓存(√)

在修改数据库的数据的时候：

  • 推荐删除Redis中的缓存
  • 修改缓存的业务复杂

操作的顺序

推荐先操作数据库再操作缓存：只有再修改数据库中的业务代码的时候会短时间出现数据不一致问题

先操作缓存再操作数据库：可以通过延迟双删的技术 删缓存=>修改数据库=>再删缓存(需要延迟一定时间防止并发)

分布式锁

setnx（底层实现）

特性：redis中有这个键的时候写入这个会返回false；

只有setnx返回true的时候是获取锁成功，在业务介绍时删除这个键用于释放锁。

！！！设置过期时间（不设置可能会导致死锁）

！！！过期时间<业务处理时间会导致锁失效

redisson

如何解决 过期时间<业务处理时间会导致锁失效 问题

  • 给锁加 uuid 使释放的时候确定是释放自己的锁
  • 通过一个子线程每隔一段观察业务释放完成，未完成延长锁过期时间

redis集群的分布式锁

集群中使用redisson时，默认只会在主节点设置锁就返回，集群内部自己同步这把锁。

如果同步锁的时候主节点挂了会导致锁失效。

解决方法：通过 red lock 设置锁在同步后才返回锁设置成功
